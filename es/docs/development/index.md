# Desarrollo

Aquí viven el flujo de contribución, las notas de arquitectura, los problemas de implementación y los puntos de extensión pensados para forks.

## Ruta recomendada

1. [Contribuyendo](/es/docs/development/contributing/) para requisitos previos, setup local, flujo de trabajo en GitHub y estado actual del proyecto.
2. [Descripción general del proyecto](/es/docs/development/project-overview/) para el resumen del sistema, el flujo de financiación, el ciclo de vida de la campaña y el mapa de código.
3. [Flujos de trabajo](/es/docs/development/workflows/) para la máquina de estados del pledge, el modelo de almacenamiento y el comportamiento de las rutas del Worker.
4. [Notas del desarrollador](/es/docs/development/developer-notes/) para detalles de implementación específicos del stack, guía sobre el modelo de contenido y gotchas.
5. [Guía para agentes y operadores](/es/docs/development/agents-operator-guide/) para la forma más segura de cambiar el repositorio sin desalinear el sitio, el Worker, el checkout o el comportamiento localizado.

## Configuración y extensión

- [Guía de personalización](/es/docs/development/customization-guide/) para la superficie soportada de `_config.yml`, design tokens, precios, shipping y controles de marca para forks.
- [Internacionalización](/es/docs/development/internationalization/) para config de locales, routing, catálogos de traducción y el flujo para agregar idiomas.
- [Embeds de campaña](/es/docs/development/campaign-embeds/) para rutas de embed alojadas, comportamiento de resize y reglas de localización.
- [Productos complementarios](/es/docs/development/add-on-products/) para el catálogo de productos de toda la plataforma, el modelo de inventario, el contrato de ejecución y el comportamiento de envío.
- [Guía para agentes y operadores](/es/docs/development/agents-operator-guide/) para invariantes del repositorio, fuentes de verdad y flujos de trabajo seguros para contributors y LLMs.

## Uso diario

Esta sección es una buena base cuando abres tu primer PR, mapeas una feature sobre la arquitectura existente o adaptas The Pool a un fork con marca propia.
